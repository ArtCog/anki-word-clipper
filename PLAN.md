# PLAN.md — Anki Word Clipper (MVP, v2 после ревью)

## Summary

Браузерное расширение (Manifest V3, Chrome + Firefox) + AnkiConnect. Без отдельного приложения, без backend-сервера. Anki Desktop — база карточек, AnkiConnect — локальный API-мост, расширение — интерфейс прямо на странице чтения.

Главный сценарий: пользователь выделяет слово или фразу, рядом появляется кнопка Anki, открывается компактная форма (или карточка добавляется мгновенно — по настройке), карточка уходит в Anki **вместе с предложением-контекстом и ссылкой на источник**, без ухода со страницы.

## Ключевые решения (v2)

1. **Свой тип заметки `Word Clipper`** — создаётся через `createModel` при первом запуске. Никакого угадывания `Basic`/«Простая»/`Einfach` (имена стандартных моделей локализованы — гадать бесконечно).
   - Поля: `Word`, `Translation`, `Context`, `Source`, `AddReverse`.
   - Шаблон 1 (всегда): `Word` → `Translation` + `Context` + `Source`.
   - Шаблон 2 (условный, обёрнут в `{{#AddReverse}}`): `Translation` → `Word`. Пустой `AddReverse` → обратная карточка не создаётся.
   - Если модель уже существует (`modelNames`) — не пересоздавать и не перезаписывать (пользователь мог кастомизировать шаблон).
2. **Флоу первого подключения** (без него расширение не заработает вообще):
   - `version` → проверка доступности AnkiConnect;
   - `requestPermission` → Anki показывает диалог разрешения для origin расширения (`chrome-extension://…` / `moz-extension://…`);
   - отказ / нет ответа → человеческая инструкция: запустить Anki, установить AnkiConnect (код аддона `2055492159`), нажать «разрешить».
3. **Дубликаты — ежедневный кейс, не ошибка.** Перед добавлением `canAddNotes`; на дубликат — статус «Уже есть в колоде» + кнопка «Добавить всё равно» (`options.allowDuplicate: true`). В мгновенном режиме — toast с той же кнопкой.
4. **Автозахват контекста.** Выделение расширяется до границ предложения (Range API, эвристика по `. ! ? …` + перенос строки, кап ~300 символов); слово внутри контекста выделяется `<b>`. Поле `Context` редактируемое в форме. `Source` = `title` + URL страницы.
5. **Два режима добавления:**
   - **Форма** (по умолчанию): кнопка → мини-форма → Enter.
   - **Мгновенный**: кнопка → сразу `addNote` (Translation пустой, контекст и источник заполнены) → toast «Добавлено».
   - Режим по умолчанию — в настройках; **Shift+клик по кнопке = противоположный режим** (быстрое переключение без настроек).
   - Пустой `Translation` разрешён всегда и молча: workflow «захвати сейчас — переведи вечером» — это фича.
6. **Обратная карточка:** чекбокс в мини-форме (значение запоминается как default); в мгновенном режиме берётся default из настроек.
7. **Контекстное меню — надёжный запасной вход** (плавающая кнопка не работает в Chrome PDF-viewer, глушится некоторыми сайтами): два пункта — «Добавить в Anki…» (форма) и «Добавить в Anki мгновенно». Плюс hotkey через `commands` (открыть форму для текущего выделения).
8. **Shadow DOM** для кнопки, формы и toast'ов — полная изоляция стилей от CSS страницы (и наоборот). Позиционирование по `range.getBoundingClientRect()`, репозиционирование при scroll/resize.
9. **Кроссбраузерность (Chrome + Firefox) с первого дня, тестирование Firefox — после стабилизации Chrome:**
   - фасад `const api = globalThis.browser ?? globalThis.chrome;`, только promise-стиль;
   - `background` с обоими ключами: `service_worker` (Chrome) + `scripts` (Firefox MV3 использует event pages);
   - `browser_specific_settings.gecko.id` в манифесте;
   - в Firefox host permissions MV3 выдаются по запросу — обработать `permissions.request` для `127.0.0.1:8765`.

## Структура

```text
anki-word-clipper/
├── manifest.json        # MV3, оба background-ключа, gecko.id
├── api.js               # фасад browser/chrome + клиент AnkiConnect (shared: подключается и в background, и в content)
├── background.js        # connection flow, contextMenus, commands, addNote-оркестрация
├── content.js           # отслеживание выделения, кнопка, мини-форма, toast (всё в Shadow DOM)
├── content.css          # минимум вне shadow (сам shadow стилизуется inline)
├── popup.html
├── popup.js             # статус соединения + настройки
├── icons/
└── README.md            # установка AnkiConnect, разрешение доступа, известные ограничения
```

## AnkiConnect (v6): используемые actions

`version`, `requestPermission`, `deckNames`, `modelNames`, `createModel`, `canAddNotes`, `addNote`.

Пример `addNote`:

```json
{
  "action": "addNote",
  "version": 6,
  "params": {
    "note": {
      "deckName": "Deutsch",
      "modelName": "Word Clipper",
      "fields": {
        "Word": "die Herausforderung",
        "Translation": "вызов, сложная задача",
        "Context": "Die größte <b>Herausforderung</b> war die Finanzierung.",
        "Source": "Beispiel-Artikel — https://example.com/artikel",
        "AddReverse": "y"
      },
      "options": { "allowDuplicate": false },
      "tags": ["word-clipper"]
    }
  }
}
```

## Хранение (`storage.local`)

- последняя выбранная колода;
- режим добавления по умолчанию (форма / мгновенный);
- default чекбокса «обратная карточка»;
- флаг «модель Word Clipper создана»;
- флаг пройденного onboarding'а (разрешение получено).

## UX

Поток «форма»:

1. Выделение текста → рядом кнопка `+ Anki` (не показывать: пустое/пробельное выделение, выделение внутри `input`/`textarea`/`contentEditable`).
2. Клик → мини-форма: `Word` заполнен, `Context` заполнен (редактируемый), колода = последняя, чекбокс «обратная» = последнее значение, **автофокус в `Translation`**.
3. `Enter` = добавить, `Esc`/`×` = закрыть.
4. Успех → toast «Добавлено в Anki», форма закрывается. Ошибка → **форма не закрывается, введённое не теряется**, статус внутри формы (+ кнопка «повторить» при недоступном Anki).

Поток «мгновенный»: выделение → клик → toast «Добавлено» (или «Уже есть в колоде» / понятная ошибка).

Поток «контекстное меню»: ПКМ на выделении → «Добавить в Anki…» / «Добавить мгновенно».

Popup расширения (минимальный, это НЕ центр управления):

- индикатор «Anki: подключено / недоступно» + кнопка «проверить»;
- режим по умолчанию, default обратной карточки;
- ссылка на README/инструкцию.

Ошибки — человеческим языком: Anki не запущен; AnkiConnect не установлен (код аддона + ссылка); доступ не разрешён (как разрешить); колода исчезла (обновить список); дубликат (особый статус, не «ошибка»).

## Известные ограничения (зафиксировать в README)

- Не работает: страницы `chrome://`, Chrome Web Store / addons.mozilla.org, Google Docs (текст на canvas — выделение не видно даже контекстному меню).
- Chrome PDF-viewer: плавающей кнопки нет; контекстное меню — по возможности.
- Карточка появится на телефоне только после синхронизации Anki ↔ AnkiWeb — расширение на это не влияет.
- Автоперевода, AI, озвучки, облака в MVP нет (осознанно). Архитектурный шов на будущее: интерфейс `translationProvider(word, context)` — в MVP не реализуется, просто не хардкодить форму против него.

## Test Plan

Проверить вручную:

- Первый запуск: Anki открыт → `requestPermission` → диалог в Anki → разрешено → модель `Word Clipper` создана (видна в Anki), карточка добавляется.
- Anki закрыт: понятная ошибка + «повторить»; ввод в форме не теряется.
- Одно слово / фраза / многострочное выделение — `Word` и `Context` корректны; умляуты и диакритика не ломаются.
- Обратная карточка: с чекбоксом заметка даёт 2 карточки, без — 1 (проверить в Anki Browse).
- Дубликат: повторное добавление того же слова → «Уже есть» → «Добавить всё равно» работает.
- Мгновенный режим: один клик — карточка в колоде с пустым переводом; Shift+клик открывает форму.
- Контекстное меню и hotkey работают там, где кнопка не появилась.
- Колода запоминается между сессиями и страницами.
- Стили изолированы: Wikipedia, новостной сайт, сайт с агрессивным CSS (Gmail/X), тёмная и светлая темы.
- Выделение в `input`/`textarea` кнопку не вызывает.
- Firefox smoke-test: установка временного аддона, connection flow, оба режима добавления.

## Milestones

1. **Каркас + соединение:** manifest, api.js, background, popup со статусом; `version` + `requestPermission` + onboarding-инструкция.
2. **Захват:** content script — выделение, кнопка, мини-форма в Shadow DOM, извлечение контекста.
3. **Добавление:** `createModel` (однократно), `deckNames`, `canAddNotes`/`addNote`, дубликаты, память колоды.
4. **Режимы и входы:** мгновенный режим, Shift+клик, контекстное меню, hotkey.
5. **Полировка:** ошибки/тексты, README (установка + ограничения), прогон тест-плана, Firefox smoke-test.

## Assumptions

- Anki Desktop установлен локально, AnkiConnect установлен и активен.
- Расширение работает только локально, слова никуда наружу не отправляются.
- Автоперевод, примеры, грамматика, озвучка, AI-пояснения — будущие версии; ценность MVP = максимально быстрый захват с контекстом, не покидая страницу.
